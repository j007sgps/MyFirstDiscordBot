# MyFirstDiscordBot (YouTube 通知小幫手)

這是一個建立給 Discord 的專屬機器人，主要用來監控 YouTube 頻道的新影片發布，並能與使用者在 Discord 內進行基礎互動！

## ✨ Bot 的主要功能
- **YouTube 新影片自動推播**：每 5 分鐘自動巡邏一次，若有新影片會自動發送到指定的 Discord 頻道，並支援發布時同時標記 `特定身分組` 的功能！
- **互動對話指令**：
  - `!最新影片`：機器人會立刻幫你抓出頻道目前最新發布的一段影片。
  - `!隨意看`：不知道想看什麼嗎？機器人會從近期的影片列表中隨機抽一部推薦給你。
  - `!help` (或 `!說明`)：喚出專屬的指令列表與機器人使用導覽。

---

## 💻 本機開發環境設定
如果你要在自己的電腦上修改與測試機器人，請先確認以下的設定：
1. **設定環境變數**：在專案目錄下建立 `.env` 檔案（請注意不要讓它上傳到 GitHub），並填入你的密碼：
   ```env
   DISCORD_TOKEN=你的機器人Token密碼
   ```
2. **安裝所需套件**：執行以下指令安裝所需套件庫：
   ```bash
   pip install -r requirements.txt
   ```
3. **啟動測試**：在終端機輸入下方指令即可看到「機器人已上線」：
   ```bash
   python bot.py
   ```

## ☁️ GCP 雲端主機：更新指令紀錄 (🌟 核心必備)
每次你在本機（自己家裡電腦）修改好程式碼，並成功 `git push` 到 GitHub 後，請打開 GCP 的黑色 SSH 終端機，**依序**貼上以下 5 句指令。

這樣就能無痛完成更新，並讓機器人繼續留在背景 24 小時值班：

```bash
# 1. 走進你的專案房間
cd MyFirstDiscordBot

# 2. 啟動虛擬沙盒環境 (注意左邊有沒有出現 (venv) 字樣)
source venv/bin/activate

# 3. 把原本還在偷偷躲著跑的舊版機器人強制關機
pkill -f bot.py

# 4. 從 GitHub 把修改好的全新程式碼拉下來 (覆蓋更新)
git pull

# 5. 讓新版的機器人重新回到背景 24 小時上班
nohup python3 -u bot.py &
```

> 💡 **進階小提點**：如果想要查看機器人的連線狀態，或是想偷看有沒有人正在亂玩機器人指令，可以隨時在終端機輸入 `cat nohup.out` 來查看它的日記本（背景輸出的紀錄檔案）喔！
